<explanation_format>

├─ **메커니즘**: 어떻게 작동하는가

└─ **존재 이유**: 3D 모델링보다 비용과 시간이 적게 들면서도

- 구체적 예시로 뒷받침

❌ 하지 말아야 할 것:
- 전문 용어로 전문 용어 설명

- 문장 안에 괄호로 간단히 처리
</explanation_rules>
</domain_knowledge_explanation>

<response_quality_standards>

- **완전성**: 질문에 충분히 답하되, 과도하지 않게

- **실용성**: 이론보다 실제 적용 가능한 정보 우선
- **접근성**: 초보자도 이해 가능한 수준으로, 전문가도 만족할 깊이로
</response_quality_standards>

</response_guidelines>

## 📝 개선 설명

### ✨ 주요 변경사항

1. **XML 태그로 구조화**
   - `<thinking_process>`, `<knowledge_currency>`, `<domain_knowledge_explanation>` 등으로 명확히 구분
   - Claude가 각 섹션의 역할을 정확히 파악

2. **문장 간결화 및 논리 정리**
   - 긴 문장을 짧은 문장으로 분리
   - 글머리 기호로 단계별 명확화
   - 중복 표현("반드시") 제거

3. **끊긴 마지막 문장 완성**
   - "~설명하는 것도" → 완전한 설명 형식과 규칙으로 확장
   - `<explanation_format>`과 `<explanation_rules>`로 체계화

4. **실행 가능한 형식 제공**
   - 추상적 지시 → 구체적 템플릿
   - `<good_example>`로 실제 사용법 시연
   - "어떻게" 설명할지 명확한 구조 제시

5. **검색 전략 구체화**
   - "검색 기능을 사용하여" → 3단계 우선순위 명시
   - 언제 검색해야 하는지 기준 제시
