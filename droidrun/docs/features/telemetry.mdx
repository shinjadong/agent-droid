# 원격 측정이 필요한 이유

원격 측정은 다음을 돕습니다:
- 가장 많이 사용되고 개선이 필요한 기능 식별
- 실제 사용량에 기반한 버그 수정 및 새 기능 우선순위 결정
- DroidRun이 다양한 환경에서 잘 작동하는지 확인

📖 **원격 측정(Telemetry)**
├─ **정의**: 소프트웨어 사용 패턴을 익명으로 수집하는 시스템
├─ **쉬운 비유**: 도로 교통량을 세듯이 어떤 기능이 많이 쓰이는지 파악
├─ **메커니즘**: 사용 통계를 익명으로 수집하여 개발 방향 결정
└─ **존재 이유**: 사용자 경험 개선을 위한 데이터 기반 의사결정

<b>개인 정보나 민감한 데이터는 수집하지 않습니다.</b> 모든 원격 측정은 엄격하게 익명화되어 있으며 모두를 위해 프레임워크를 개선하는 데만 사용됩니다.

질문이나 우려 사항이 있으시면 [GitHub](https://github.com/droidrun/droidrun)로 문의하거나 개인정보 보호 정책을 검토하세요.

# 원격 측정 토글

DroidRun은 <b>익명화된</b> 사용 데이터를 수집하여 어떤 기능이 가장 가치 있고 어디에 개선이 필요한지 파악합니다. 이 데이터는 광고나 개인 추적에 <b>절대</b> 사용되지 않으며 커뮤니티를 위해 DroidRun을 더 나아지게 만드는 데만 사용됩니다.

## 원격 측정 비활성화 방법

다음 환경 변수를 설정하여 언제든지 원격 측정을 비활성화할 수 있습니다:

```bash
export DROIDRUN_TELEMETRY_ENABLED=false
```

세션 간 지속성을 위해 셸 프로필(예: `.bashrc`, `.zshrc`, `.profile`)에 이 줄을 추가하세요.

## 원격 측정 다시 활성화 방법

원격 측정을 다시 활성화하려면 환경 변수를 `true`로 설정하세요:

```bash
export DROIDRUN_TELEMETRY_ENABLED=true
```

또는 환경에서 변수를 제거하여 기본(활성화) 동작을 사용하세요.

```sh
echo 'export DROIDRUN_TELEMETRY_ENABLED=false' >> ~/.bashrc
source ~/.bashrc
```

<Note>
원격 측정은 현재 `DROIDRUN_TELEMETRY_ENABLED` 환경 변수로만 제어됩니다. `telemetry.enabled` 설정 옵션은 아직 구현되지 않았습니다.
</Note>
